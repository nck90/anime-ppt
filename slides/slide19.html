Git & GitHub 기술 면접 준비

기술 면접에서 자주 등장하는 Git 관련 질문과 효과적인 대응 전략

Git과 GitHub의 차이점은?

면접에서 기본적으로 물어보는 질문으로, Git과 GitHub의 개념적 차이를 명확하게 설명할 수 있어야 합니다.

모범 답변: Git은 분산 버전 관리 시스템으로 로컬에서 코드 변경 사항을 추적하는 도구이며, GitHub는 Git 저장소를 호스팅하는 클라우드 기반 서비스로 협업 기능을 제공합니다.

Git 브랜치 전략을 설명해주세요

실무 경험을 확인하는 질문으로, 브랜치 전략에 대한 이해와 실제 적용 경험을 보여주세요.

모범 답변: Git Flow와 GitHub Flow 두 가지 주요 전략이 있습니다. 복잡한 프로젝트에는 master, develop, feature, release, hotfix 브랜치를 활용하는 Git Flow가 적합하고, 지속적 배포에는 단순한 GitHub Flow가 효과적입니다.

Rebase와 Merge의 차이점은?

Git의 고급 기능에 대한 이해도를 확인하는 질문으로, 각각의 장단점과 적절한 사용 상황을 설명하세요.

모범 답변: Merge는 브랜치의 변경사항을 통합하여 병합 커밋을 생성하며 이력이 보존됩니다. Rebase는 커밋을 재배치하여 선형적인 이력을 만들어 깔끔한 히스토리를 제공하지만, 공개 브랜치에서는 사용을 주의해야 합니다.

Git에서 충돌을 해결한 경험이 있나요?

실무 경험과 문제 해결 능력을 평가하는 질문으로, 구체적인 사례와 해결 방법을 공유하세요.

모범 답변: 팀 프로젝트에서 동일 파일을 수정하여 충돌이 발생했습니다. git status로 충돌 파일을 확인하고, 코드 에디터에서 충돌 마커(<<<<<<<, =======, >>>>>>>)를 찾아 양쪽 변경사항을 검토한 후 의도에 맞게 통합했습니다.

실전 면접 시나리오

면접관

팀에서 Git으로 협업할 때 다른 개발자가 작성한 코드와 충돌이 발생했다면 어떻게 해결하시겠습니까?

지원자 (모범 답변)

먼저 git pull로 최신 변경사항을 가져온 후, 충돌이 발생한 파일을 확인합니다. 충돌 부분을 검토하고 필요하다면 해당 코드를 작성한 팀원과 직접 소통하여 의도를 확인합니다. 충돌을 해결한 후에는 코드가 정상 작동하는지 테스트하고, 그 다음 변경사항을 커밋합니다. 충돌이 자주 발생한다면 브랜치 전략을 재검토하거나 더 작은 단위로 자주 커밋하는 방식을 팀에 제안할 것입니다.

Pro Tip:  면접 중 GitHub 프로필을 보여줄 때는 미리 특정 코드나 커밋 이력으로 이동할 수 있도록 북마크를 준비해두면 설명이 더 매끄러워집니다.
