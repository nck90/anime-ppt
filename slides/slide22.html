자주 묻는 질문 (FAQ)

Git, GitHub 및 백준허브 사용 시 자주 나오는 질문과 답변

백준허브가 작동하지 않을 때는 어떻게 해야 하나요?

백준허브가 작동하지 않는 경우 다음 방법을 시도해보세요:

  • Chrome 확장 프로그램 페이지에서 백준허브를 비활성화했다가 다시 활성화
  • GitHub 계정 재인증 진행
  • 백준 사이트에서 solved.ac 연동 확인
  • 크롬 브라우저 재시작
  • 최신 버전으로 업데이트

Git 커밋을 실수로 했을 때 되돌리는 방법은?

실수로 한 커밋을 되돌리는 여러 방법이 있습니다:

# 가장 최근 커밋 취소 (변경사항 유지)
git reset --soft HEAD~1

# 완전히 되돌리기 (변경사항 삭제)
git reset --hard HEAD~1

# 새 커밋으로 이전 커밋 되돌리기
git revert HEAD

공개 저장소의 경우 revert가 더 안전한 방법입니다.

백준허브와 GitHub 연동 시 여러 기기에서 사용할 수 있나요?

네, 여러 기기에서 백준허브를 사용할 수 있습니다. 각 기기에 백준허브를 설치하고 동일한 GitHub 계정으로 인증하면 됩니다. 이 경우 모든 기기에서의 문제 풀이가 동일한 GitHub 저장소에 기록됩니다.

주의사항:

서로 다른 기기에서 동시에 같은 문제를 풀 경우 Git 충돌이 발생할 수 있습니다.

Git과 SVN의 차이점은 무엇인가요?

Git

  • 분산형 버전 관리 시스템
  • 전체 리포지토리 복제
  • 오프라인 작업 가능
  • 빠른 브랜칭과 병합

SVN

  • 중앙집중식 버전 관리 시스템
  • 일부 파일만 체크아웃 가능
  • 작업에 네트워크 연결 필요
  • 더 단순한 학습 곡선

백준허브의 커밋 메시지 형식을 변경할 수 있나요?

네, 백준허브의 설정에서 커밋 메시지 템플릿을 사용자 정의할 수 있습니다:

  1. 백준허브 확장 프로그램 아이콘 클릭
  2. 설정(⚙️) 아이콘 클릭
  3. 커밋 메시지 템플릿 섹션에서 형식 변경

다음과 같은 변수를 사용할 수 있습니다: {문제번호}, {문제이름}, {난이도}, {언어} 등

GitHub 리포지토리를 클론할 때 큰 파일이 있으면 어떻게 처리하나요?

대용량 파일이 있는 리포지토리를 효율적으로 다루기 위해서는 Git LFS(Large File Storage)를 사용하세요:

# Git LFS 설치
$ git lfs install

# 대용량 파일 추적 설정
$ git lfs track "*.bin" "*.psd" "*.zip"

# .gitattributes 파일 커밋
$ git add .gitattributes

이 설정 후 push와 pull 시 해당 확장자의 파일들은 Git LFS를 통해 효율적으로 관리됩니다.
